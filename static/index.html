REST placeholder endpoints

Here are described the existing endpoints for this RESTful API. The authentication is achieved by using an "X-Auth-Key" header in the request (you receive this authKey upon sign up/sign in).

Users

GET /users

Responds with the user's profile

GET /users/:username

Responds with an user's profile

POST /users

Registers a new user, requires { username, passHash, passHashRepeat }, responds with { username, authKey }.

PUT /users

Authenticates an user, requires { username, passHash }, responds with { username, authKey }.

Posts

GET /posts

Responds with all posts

POST /posts

Creates a new post, responds with the new post, requires header "X-Auth-Key": authKey

GET /posts/id/:id

Responds with a post with certain id

PUT /posts/category/:category

Responds with posts in a certain category
