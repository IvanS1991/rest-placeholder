REST placeholder endpoints

Here are described the existing endpoints for this RESTful API. The authentication is achieved by using an "X-Auth-Key" header in the request (you receive this authKey upon sign up/sign in).

Users

GET /users

Responds with the user's profile, requires header "X-Auth-Key": authKey

GET /users/:username

Responds with an user's profile

POST /users

Registers a new user, requires { username(string), passHash(string), passHashRepeat(string) }, responds with { username, authKey }.

PUT /users

Authenticates an user, requires { username(string), passHash(string) }, responds with { username, authKey }.

Posts

GET /posts

Responds with all posts

POST /posts

Creates a new post, responds with the new post, requires header "X-Auth-Key": authKey

GET /posts/:id

Responds with a post with certain id

PUT /posts/category/:category

Responds with posts in a certain category

DELETE /posts/:id

Deletes a post and its comments

Comments

POST /comments

Creates a new comment, requires { postId(string), content(string) }, responds with the new comment.

DELETE /comments/:postId/:commentId

Deletes a comment

TODOs

POST /todos

Creates a new TODO, requires { description(string) }, responds with the new TODO.

GET /todos

Responds with the currently logged in user's TODOs

PUT /todos/:id

Updates a TODO with this id, accepts { ?description(string), ?completed(boolean) }, responds with the updated TODO.
